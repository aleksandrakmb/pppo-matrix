## Następne kroki — od czego zacząć za tydzień

### Krok 1 — Faza 1: Zarządzanie sesjami
To jest kolejny feature do zbudowania. PPO po wyborze roli trafia na listę sesji — która jest teraz placeholderem. Trzeba ją ożywić.

**Co zbudować (US-002):**
- Lista sesji na `/sessions` — pusta lub z zapisanymi sesjami
- Przycisk "Utwórz nową sesję"
- Formularz: nazwa sesji
- Sesje zapisywane w localStorage
- Usuwanie sesji

**Pliki do stworzenia:**
```
src/lib/storage.ts        ← rozszerzyć o funkcje dla sesji (getSessions, createSession, deleteSession)
src/app/sessions/page.tsx ← zastąpić placeholder prawdziwą listą
src/types/index.ts        ← dodać typ Session
```

### Krok 2 — Przed kodem: opisz US-002
Tak jak dziś zrobiłyśmy US-001 — najpierw user story z acceptance criteria, potem kod.

## Jak zacząć za tydzień

1. Otwórz terminal w folderze projektu
2. `git checkout dev` — upewnij się że jesteś na branchu dev
3. `npm run dev` — odpala aplikację lokalnie na localhost:3000
4. Powiedz Claude Code: _"wracamy do projektu, podsumuj gdzie skończyłyśmy"_ — Claude przeczyta ten plik i będzie gotowy
